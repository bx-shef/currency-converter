# Публикация в Маркет Bitrix24 — гейты релиза

> Last reviewed: 2026-07-23

Чек-лист перед публикацией/обновлением приложения в Маркете Bitrix24. Цель — не выпустить
в Маркет то, что сломано в портале (install/widget нельзя проверить автотестами — нужен
реальный портал). Полный ручной сценарий — в [`MANUAL_TEST_PLAN.md`](MANUAL_TEST_PLAN.md).

## Гейты (все обязательны)

| # | Гейт | Как проверить | ✓ |
|---|------|---------------|---|
| 1 | Автопроверки зелёные | `pnpm check` (lint + typecheck + test) — те же, что в CI | |
| 2 | Прод задеплоен и жив | `make prod-smoke-external` → `200` (см. [`OPERATIONS.md`](OPERATIONS.md)) | |
| 3 | Визуальная проверка | `pnpm screenshots` → просмотреть `/`, `/widget/converter`, `/install` в свете/тьме на 375/1280 (см. [`VISUAL_VERIFICATION.md`](VISUAL_VERIFICATION.md)) | |
| 4 | Ручной прогон в портале | `MANUAL_TEST_PLAN.md` разделы 5–7: install → bind плейсмента `IM_TEXTAREA` → widget «Вставить в чат» → dual-mode главной | |
| 5 | Версия сборки | подвал показывает «сборка `<sha>`», ссылается на актуальный коммит (`utils/build.ts`) | |
| 6 | `MARKETPLACE_URL` актуален | листинг в `utils/site.ts` (`MARKETPLACE_URL`) / env `NUXT_PUBLIC_MARKETPLACE_URL` указывает на опубликованное приложение | |
| 7 | Имя приложения на всех языках | `app.title` во всех локалях (`i18n/locales/*.json`) — `LANG_ALL` берёт имя чип-плейсмента **в момент install** | |
| 8 | CSP разрешает облако Б24 | `nginx.conf`: `frame-ancestors`/`connect-src` включают `*.bitrix24.*` (self-hosted домены — вручную) | |
| 9 | Приватность | [`DATA_POLICY.md`](DATA_POLICY.md) актуален (для карточки Маркета / вопросов о данных) | |

## Особые случаи

- **После major-бампа `@bitrix24/b24jssdk`** — рантайм SDK автотесты не покрывают (в тестах
  `useB24` мокается). **Обязателен** ручной прогон в реальном портале: `/install` → bind
  `IM_TEXTAREA` → `/widget/converter` (Insert в поле ввода чата). См. `CLAUDE.md` (раздел встройки).
- **Смена `app.title`** — уже установленным порталам нужна **переустановка** приложения, чтобы
  подхватить новое имя на чип-плейсменте (имя фиксируется в момент install). Учесть в описании
  обновления для пользователей.
- **Смена домена/`NUXT_PUBLIC_SITE_URL`** — `install.vue` биндит абсолютный HANDLER; при смене
  домена старые привязки указывают на старый URL → нужна переустановка. Пересобрать образ с
  новым build-arg (значение запекается в SSG на `generate`).

## Порядок

1. Смержить изменения в `main` → дождаться авто-деплоя (Watchtower ~5 мин) или `make prod-redeploy`.
2. Пройти гейты 1–9 выше.
3. Обновить листинг в кабинете разработчика Маркета (описание/скриншоты/версия).
4. После публикации — проверить установку из Маркета на тестовом портале (гейт 4 ещё раз, «из Маркета»).

## Известные ограничения (не блокеры публикации, задокументированы)

- Мобильное приложение Б24: виджет view-only, «Вставить в чат» скрыт (issue #124).
- Ввод разделителя точкой в `B24InputNumber` (issue #136) и подстройка под ресайз фрейма (issue
  #138) — поведение портала/компонента, отслеживается отдельно.
