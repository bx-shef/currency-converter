# Визуальная проверка (скриншот-харнесс)

> Last reviewed: 2026-07-22

Скрипт `scripts/screenshots.mjs` снимает каждую страницу в матрице **роут × вьюпорт × тема**,
чтобы UI-правку можно было глазами сверить в светлой/тёмной теме на мобиле и десктопе — без
ручного клика по портам и devtools. Это dev-инструмент: он **не** гоняется в CI (не хотим гонять
браузер и ловить флаки на каждый PR) и не влияет на прод-сборку.

## Матрица

| Ось | Значения |
|---|---|
| Роут | `/` (главная), `/widget/converter` (виджет чата), `/install` (обработчик установки) |
| Вьюпорт | `mobile` 375×812, `desktop` 1280×900 (оба — `deviceScaleFactor: 2`, ретина) |
| Тема | `light`, `dark` (через эмуляцию `prefers-color-scheme` → auto-colorMode приложения) |

Итого 3 × 2 × 2 = **12 PNG** в `screenshots/` (каталог в `.gitignore`).

## Как запустить

Сервер и «снимальщик» намеренно разделены (надёжнее и кроссплатформенно). Сначала поднять
статику, потом снять:

```bash
pnpm generate            # собрать SSG в .output/public
pnpm preview             # отдать на http://localhost:3000 (терминал 1)
pnpm screenshots         # снять скриншоты (терминал 2) → screenshots/*.png
```

Переменные окружения:

- `BASE_URL` — адрес запущенного приложения (по умолчанию `http://localhost:3000`).
- `PLAYWRIGHT_CHROMIUM_PATH` — явный путь к бинарю Chromium. Если не задан, скрипт
  находит его через `playwright-core` (штатный путь на любой ОС — `playwright install chromium`)
  или в бандле под `PLAYWRIGHT_BROWSERS_PATH` (так Chromium предустановлен в образах
  Claude Code / CI; резолв бинаря — по `process.platform`, берётся максимальная ревизия).
  На macOS/Windows, если резолв не сработал, задайте `PLAYWRIGHT_CHROMIUM_PATH` явно.

## Зачем именно так

- **`playwright-core` + `executablePath`** — берём уже установленный Chromium, не тянем свой
  бандл браузера в `node_modules` (лишние ~150 МБ) и не зависим от точной ревизии.
- **`colorScheme` вместо инъекции `localStorage`** — тема `auto` приложения следует за
  `prefers-color-scheme`, поэтому свет/тьму задаём опцией контекста, без хаков со стором.
- **Курсы НБ РБ** в песочнице/оффлайн не загрузятся (`api.nbrb.by` недоступен) — страница
  покажет ошибку загрузки. Это ожидаемо: харнесс проверяет **вёрстку/тему/промо**, а не
  живые курсы; `networkidle` при зависшем фиде мягко откатывается на ожидание `load`.

## Что смотреть на скриншотах

- Главная (`/`): шапка, блок курсов (или ошибка загрузки), промо-карточки
  (`AppInBitrixCard` cyan, `CustomDevCard` copilot-градиент), подвал.
- Виджет (`/widget/converter`): компактная раскладка под узкий iframe, «прописью», переключатель
  регистра «аб/Аб», кнопка «Вставить в чат».
- Установка (`/install`): вне iframe снимок ловит экран мок-прогресса установки
  («Application installation / Loading portal data…» + сворачиваемый «Diagnostics») —
  standalone-редирект на `/` происходит позже (~через паузы мок-шагов) и в кадр не попадает.
