# Политика данных и диагностики

> Last reviewed: 2026-07-23

Что приложение делает с данными и что именно оно измеряет. Объединяет «потоки данных»
(P2-3) и «диагностику» (P2-4) в один документ — для статического виджета без бэкенда
это одна тема, дробить на два дока смысла нет.

**Коротко:** конвертер — чистая статика (SSG), без своего сервера, БД и логов. Единственные
исходящие вызовы — публичные курсы НБ РБ и (только standalone) Яндекс.Метрика. Инвариант
телеметрии — **«shape/outcome, never content»**: наружу уходит только факт/бакет события,
никогда сами значения (суммы, курсы, ввод пользователя).

## 1. Что покидает браузер (сеть)

| Куда | Что | Данные пользователя? |
|---|---|---|
| `api.nbrb.by` (GET `exrates/rates`, дневной + месячный фид) | публичные курсы НБ РБ | Нет — только читаем курсы |
| Яндекс.Метрика (`mc.yandex.ru/metrika/tag.js`) — **только standalone** | обезличенная веб-аналитика + цели (см. §3) | Нет значений/ввода (см. инвариант) |

Больше исходящих каналов нет: приложение отдаётся как статика за nginx, своего API/бэкенда
у него нет. Внутри портала Б24 (iframe) Метрика **не грузится вовсе** (см. §4).

**Про Яндекс.Метрику отдельно.** Счётчик инициализируется с `clickmap: true`, `trackLinks: true`,
`accurateTrackBounce: true` (`public/metrika.js`). Это **встроенные** фичи Метрики (карта кликов,
трекинг переходов по ссылкам, точный показатель отказов) — они работают независимо от наших
целей из §3 и находятся **вне нашего периметра** «shape/outcome» (инвариант §3 — про данные
приложения: суммы/курсы/ввод туда не попадают и через встроенные фичи тоже). Метрика ставит
**свои cookies** и обрабатывает данные по своей политике; всё это — **только standalone**, в
портале Б24 Метрика заглушена (§4).

## 2. Что хранится локально (`localStorage`, не покидает браузер)

| Ключ | Что | Зачем | Срок |
|---|---|---|---|
| `nbrb_rates_v2` | кэш курсов НБ РБ (коды + `bynRate` + дата + timestamp) | меньше запросов, работа при кратком офлайне | TTL 12 ч |
| `converter_helpful_v1` | факт ответа на 👍/👎 «Помог курс?» (`'1'`) | не спрашивать повторно | бессрочно |
| `vueuse-color-scheme` | выбор темы (light/dark/auto) | сохранить тему между визитами | бессрочно |

Всё это остаётся в браузере пользователя и никуда не отправляется. Значения сумм/ввода
в `localStorage` **не** сохраняются.

## 3. Диагностика — цели Яндекс.Метрики (только standalone)

**Инвариант:** шлём **факт** или **бакет** события, а не содержимое. Никаких сумм, курсов,
текста ввода, сумм прописью. Имена целей — фиксированные константы.

| Цель | Когда срабатывает | Источник |
|---|---|---|
| `rates_load_failed` | сбой загрузки дневного (основного) фида курсов | `useNbrbRates` |
| `rates_monthly_missing` | дневной фид успешен, месячный (best-effort) упал | `useNbrbRates` |
| `converter_helpful_yes` / `converter_helpful_no` | клик 👍/👎 в блоке «Помог курс?» | `index.vue` |
| `market_card_click` | клик CTA карточки «Приложение для Bitrix24» | `AppInBitrixCard` |
| `market_qr_reveal` | показ QR листинга Маркета (hold-to-reveal) | `HoldRevealQr` |
| `custom_dev_click` | клик CTA карточки «Нужна доработка…» | `CustomDevCard` |
| `custom_dev_qr_reveal` | показ QR оффера (hold-to-reveal) | `HoldRevealQr` |
| `web_vitals_<lcp\|cls\|inp>_<good\|ni\|poor>` | замер Core Web Vitals → бакет по рейтингу (не мс) | `webVitals.client.ts` |

Чистые ядра целей покрыты юнит-тестами (`tests/metrika.test.ts`, `tests/webVitals.test.ts`,
`tests/site.test.ts`); health-цели курсов — `tests/nuxt/composables.nuxt.test.ts`.

## 4. Приватность внутри портала Bitrix24

В iframe портала телеметрии **нет вообще** — двойная защита:

1. `public/metrika.js` глушит себя во фрейме (`window.self !== window.top`) — Метрика не
   инициализируется, `window.ym` не появляется.
2. Плагины/цели гейтятся предикатом `isEmbedded` (`app/utils/isEmbedded.ts`): web-vitals-плагин
   даже не грузит библиотеку в портале; `reachMetrikaGoal` — no-op при незагруженной Метрике.

То есть пользователей портала мы не трекаем by design (осознанное решение — см. `CLAUDE.md`,
раздел про `app/layouts/default.vue`).

## 5. Чего НЕТ

- Нет своего сервера/БД/бэкенд-логов действий пользователя.
- Нет отправки сумм/курсов/ввода/сумм прописью — ни в Метрику, ни куда-либо ещё.
- Нет сторонних скриптов, кроме Яндекс.Метрики (и только standalone).
- Нет собственных cookies; cookies ставит только Яндекс.Метрика (standalone), по своей политике.

## 6. Как посмотреть/проверить

- Список активных целей — в отчёте Яндекс.Метрики (счётчик задаётся `NUXT_PUBLIC_YANDEX_COUNTER_ID`).
- Ручная проверка целей — `docs/MANUAL_TEST_PLAN.md` §8.
- Пустой/невалидный счётчик или отсутствие Метрики → все цели no-op (`utils/metrika.ts`).
